# حالة تطبيق الموبايل — BI Management Mobile

---

## الربط بالـ API المعتمد

- التطبيق **مرتبط بـ Backend النظام الحالي** (Node/Express).
- في التطوير: `API_URL = 'http://127.0.0.1:3000/api'` (مع استخدام adb reverse للجهاز الفعلي).
- في الإنتاج: `API_URL = 'https://api.bi-management.com/api'` (يُعدّل حسب عنوان الـ API الفعلي).

---

## ما يعمل على الموبايل (حسب الشاشات والخدمات)

| الميزة | الشاشة/الملف | الحالة |
|--------|--------------|--------|
| تسجيل الدخول | LoginScreen، AuthContext | متصل بـ `/api/auth/login` و token |
| الرئيسية | HomeScreen | لوحة رئيسية |
| المهام | TasksScreen، TaskDetailsScreen | قائمة مهام وتفاصيل (يفترض `/api/tasks`) |
| الحضور | AttendanceScreen | حضور/انصراف (يفترض `/api/attendance`) |
| الإشعارات | NotificationsScreen | إشعارات (يفترض `/api/notifications`) |
| الدردشة | ChatScreen | دردشة |
| الأجهزة/المسح | DeviceDetailsScreen، ScanScreen | تفاصيل جهاز ومسح (يفترض `/api/devices` أو مسار مسح) |

---

## الميزات الحرجة للمدير (مبيعات، حضور، مهام)

- **الحضور:** شاشة مخصصة — يفترض أنها تعمل مع مسارات الحضور في الـ Backend الحالي.
- **المهام:** شاشة مهام وتفاصيل — يفترض أنها تعمل مع `/api/tasks`.
- **المبيعات (فواتير):** لا توجد شاشة مخصصة لإنشاء فواتير في الموبايل ضمن الملفات المذكورة؛ إن احتجت مبيعات من الموبايل يلزم إضافة شاشة/ربط مع `/api/invoices`.

---

## التوصيات

1. **التحقق:** تشغيل التطبيق (Expo) والاتصال بالـ Backend الحالي والتحقق من تسجيل الدخول والمهام والحضور فعلياً.
2. **عنوان الإنتاج:** تحديث `API_URL` في `src/services/api.js` ليطابق عنوان الـ API المعتمد في بيئة الإنتاج.
3. **إضافة ميزات:** إن رُغب بإنشاء فواتير أو عرض تقارير من الموبايل، إضافة الشاشات وربطها بمسارات الـ API الحالية (`/api/invoices`، `/api/reports/my-sales`، إلخ).
